# D2RE Visual Workbench

The D2RE Visual Workbench is the first GUI layer for the toolkit. It is a local, standalone browser interface generated by `d2re gui`.

The first version is intentionally read-only. It helps users discover workflows, copy known-good commands, and understand which subsystem to use without adding a long-running service, database, or unsafe file mutation path.

## Launch

Generate and open the workbench:

```bash
d2re gui
```

Generate without opening a browser:

```bash
d2re gui --no-open --print-path
```

Write to a chosen path:

```bash
d2re gui --out ./d2re-workbench.html --no-open
```

A dedicated console entry point is also available after package installation:

```bash
d2re-gui --no-open --print-path
```

## What it includes

| Module | Purpose |
|---|---|
| Save Codex | Start `.d2s` parsing workflows. |
| Item Forge | Start deterministic item roll and Magic Find workflows. |
| Treasure Labyrinth | Start Treasure Class inspection workflows. |
| Drop Oracle | Start Monte Carlo drop-estimation workflows. |
| Map Observatory | Start map seed and DRLG workflows. |
| Packet Timeline | Start packet demo and decode workflows. |
| Data Excavator | Start MPQ/CASC extraction workflows. |
| Repository Doctor | Shows the planned diagnostic surface while `d2re doctor` remains disabled. |

## UX and design decisions

This GUI applies the uploaded UI/UX skill pack as a practical implementation checklist rather than as decorative frosting.

### Accessibility

- Visible skip link for keyboard users.
- Sequential headings.
- Full keyboard focus visibility with a high-contrast focus ring.
- Controls meet a 44px minimum interaction target.
- Toast output uses `role="status"` and `aria-live="polite"`.
- Motion respects `prefers-reduced-motion`.

### Layout

- Responsive two-column desktop layout.
- Single-column layout on smaller screens.
- Sticky module navigation on large screens.
- Search and status filters for progressive disclosure.

### Visual system

- Tokenized CSS variables for primitives, semantic surfaces, focus, spacing, radius, and typography.
- Dark gothic treatment with obsidian surfaces, blood-red accents, parchment text, and metal-like panel borders.
- No external images, fonts, frameworks, or third-party scripts.

### Safety

- The GUI does not execute commands.
- The GUI does not edit save files.
- The GUI does not start packet capture.
- The GUI only writes a standalone HTML file and optionally opens it in the default browser.

## Implementation notes

The workbench lives in `d2re/gui.py`.

Main functions:

| Function | Purpose |
|---|---|
| `build_model()` | Builds the card data displayed by the workbench. |
| `render_workbench()` | Renders standalone HTML. |
| `write_workbench()` | Writes the HTML file. |
| `main()` | CLI entry point. |

Smoke tests live in `tests/test_gui_workbench.py`.
